We use a colour design system to define how our colours are used
1. **Primary Colours** - used very frequently to create the main experience - includes tints, shades, and tones of primary colours
2. **Secondary Colours** - used occasionally, usually for functions like errors, success, or warning
- Can be defined using CSS hex, name, rgb, hsl

**Typography**
- has terminology based on how we want to format the text
![[Pasted image 20250408141313.png]]

**Typeface Styles:**
- Serif, sans-serif, display, script 
- Usually prefer simple standard typefaces (avoid digital and script)
- Can define size, weight, and emphasis to change the font

**Alignment**
- left alignment for paragraphs
- left or center for headings
- center for buttons/interface controls
- **never** use fully justified - hard to read and leaves rivers


**Layout** - The structure that organizes visual elements in an interface
**Grid** - Underlying regularized structure to organize elements in a layout
- usually 12 columns (divisible easily) 
- often has a baseline grid to place text

**8-point pixel grid** uses multiples of 8 to define dimensions, padding, and margins
**Negative Space** is the space between content (white space) - just no content
- groups and divides content
- lets the users eye rest
- created by margin and padding

**Alignment** - when the dominant visual parts of multiple elements form an implied line
- must be done on visual parts of an element (not just bounding box) 
- using a grid makes good alignments

**Use `@media` query to change the grid layout for mobile and desktop**
